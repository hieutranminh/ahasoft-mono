# Internationalization (i18n) Documentation

This document outlines the i18n implementation using vue-i18n v9, TypeScript, and Composition API.

## Architecture Overview

```
src/
├── locales/
│   ├── en.ts                    # English translations
│   ├── vi.ts                    # Vietnamese translations
│   ├── ko.ts                    # Korean translations
│   └── index.ts                 # Exports, types, config
├── plugins/
│   └── i18n.ts                  # vue-i18n instance + setI18nLanguage()
├── composables/
│   └── useI18n.ts               # i18n composable (locale management)
└── components/
    └── common/
        └── LanguageSwitcher.vue # UI component
```

## Key Features

### Type-Safe Locale Management

- `Locale` type derived from `availableLocales` array
- TypeScript ensures only valid locales are used
- Adding new languages auto-updates the `Locale` type

### Self-Contained i18n Module

- `useI18n.ts` composable manages all locale logic
- No dependency on auth store for locale state
- Uses `sessionStorage` directly for persistence

### Auth Integration

- After login, backend `language` field (e.g. `"EN"`, `"KO"`) is mapped to app locale
- `mapBackendLanguageToLocale()` converts backend codes to app locales
- `persistLocale()` updates vue-i18n and saves to sessionStorage

## Core Files

### 1. Locale Configuration (`src/locales/index.ts`)

```typescript
import en from './en'
import vi from './vi'
import ko from './ko'

export const messages = { en, vi, ko }

export const availableLocales = ['en', 'vi', 'ko'] as const

export type Locale = (typeof availableLocales)[number]

export const defaultLocale: Locale = 'en'
```

### 2. i18n Plugin (`src/plugins/i18n.ts`)

```typescript
import { createI18n } from 'vue-i18n'
import { messages, defaultLocale, type Locale } from '@/locales'

export const i18n = createI18n({
  legacy: false,
  locale: defaultLocale,
  fallbackLocale: defaultLocale,
  messages,
  globalInjection: true,
  missingWarn: false,
  fallbackWarn: false,
})

export const setI18nLanguage = (locale: Locale): void => {
  i18n.global.locale.value = locale
  if (typeof document !== 'undefined') {
    document.querySelector('html')?.setAttribute('lang', locale)
  }
}
```

- `legacy: false` for Composition API
- `globalInjection: true` allows `$t()` in templates without imports
- `setI18nLanguage` is internal, called only from `persistLocale()`

### 3. useI18n Composable (`src/composables/useI18n.ts`)

```typescript
import { computed } from 'vue'
import { useI18n as vueUseI18n } from 'vue-i18n'
import { setI18nLanguage } from '@/plugins/i18n'
import { availableLocales, defaultLocale, type Locale } from '@/locales'

const LOCALE_STORAGE_KEY = 'app_locale'

const BACKEND_LANGUAGE_TO_LOCALE: Record<string, Locale> = {
  EN: 'en',
  KO: 'ko',
  VI: 'vi',
}

export function mapBackendLanguageToLocale(backendLanguage: string): Locale {
  return BACKEND_LANGUAGE_TO_LOCALE[backendLanguage.toUpperCase()] ?? defaultLocale
}

export function getStoredLocale(): Locale | null {
  const stored = sessionStorage.getItem(LOCALE_STORAGE_KEY)
  if (stored && (availableLocales as readonly string[]).includes(stored)) {
    return stored as Locale
  }
  return null
}

export function persistLocale(locale: Locale): void {
  setI18nLanguage(locale)
  sessionStorage.setItem(LOCALE_STORAGE_KEY, locale)
}

export function useI18n() {
  const i18n = vueUseI18n()

  const changeLocale = (newLocale: Locale): void => {
    persistLocale(newLocale)
  }

  return {
    ...i18n,
    changeLocale,
    currentLocale: computed(() => i18n.locale.value as Locale),
  }
}
```

**Exported functions:**

| Function                       | Purpose                                           | Used by                    |
| ------------------------------ | ------------------------------------------------- | -------------------------- |
| `mapBackendLanguageToLocale()` | Convert backend code to app locale                | `LoginView.vue`            |
| `getStoredLocale()`            | Read locale from sessionStorage                   | `main.ts`                  |
| `persistLocale()`              | Update vue-i18n + sessionStorage                  | `LoginView.vue`, `useI18n` |
| `useI18n()`                    | Composable for components (t, changeLocale, etc.) | Any Vue component          |

## Usage Examples

### 1. In Templates (Simplest)

```vue
<template>
  <div>
    <h1>{{ $t('common.welcome') }}</h1>
    <Button :label="$t('common.save')" />
    <p>{{ $t('validation.required', { field: 'Email' }) }}</p>
  </div>
</template>
```

### 2. With Composition API

```vue
<script setup lang="ts">
import { useI18n } from '@/composables/useI18n'

const { t, changeLocale, currentLocale } = useI18n()

const title = t('common.welcome')
const switchToKorean = () => changeLocale('ko')
console.log(currentLocale.value) // 'en', 'vi', or 'ko'
</script>
```

### 3. After Login (Setting Language from Backend)

```typescript
import { mapBackendLanguageToLocale, persistLocale } from '@/composables/useI18n'

// In LoginView.vue after successful login
const locale = mapBackendLanguageToLocale(response.result.userAuthInfo.language)
persistLocale(locale)
```

### 4. LanguageSwitcher Component

```vue
<template>
  <Select
    v-model="selectedLocale"
    :options="localeOptions"
    option-label="label"
    option-value="value"
  />
</template>

<script setup lang="ts">
import { computed } from 'vue'
import { useI18n } from '@/composables/useI18n'
import type { Locale } from '@/locales'

const { changeLocale, currentLocale } = useI18n()

const localeOptions = [
  { label: 'English', value: 'en' },
  { label: 'Tiếng Việt', value: 'vi' },
  { label: '한국어', value: 'ko' },
]

const selectedLocale = computed({
  get: () => currentLocale.value,
  set: (value: Locale) => changeLocale(value),
})
</script>
```

## Adding New Content

### Adding Translation Keys

1. Add to all locale files (`en.ts`, `vi.ts`, `ko.ts`).
2. Use in component: `$t('module.key')`.

### Adding New Language

1. Create translation file `src/locales/ja.ts`.

2. Update `src/locales/index.ts`:

```typescript
import en from './en'
import vi from './vi'
import ko from './ko'
import ja from './ja'

export const messages = { en, vi, ko, ja }
export const availableLocales = ['en', 'vi', 'ko', 'ja'] as const
// Locale type auto-updates to 'en' | 'vi' | 'ko' | 'ja'
```

3. Add backend mapping in `src/composables/useI18n.ts`:

```typescript
const BACKEND_LANGUAGE_TO_LOCALE: Record<string, Locale> = {
  EN: 'en',
  KO: 'ko',
  VI: 'vi',
  JA: 'ja', // new
}
```

4. Update `LanguageSwitcher.vue` options.

## Best Practices

### Do's

- Use `useI18n()` composable in components
- Use `persistLocale()` when changing locale outside components
- Use `mapBackendLanguageToLocale()` to convert backend language codes
- Keep translation keys organized by module
- Use parameters for dynamic content: `$t('validation.minLength', { field: 'Password', min: 8 })`
- Sync all keys between language files

### Don'ts

- Don't use `setI18nLanguage()` directly (internal, use `persistLocale()`)
- Don't access `i18n.global.locale` directly
- Don't use flat key structure (use nested modules)
- Don't concatenate translated strings
- Don't import `useI18n` from `vue-i18n` directly (use `@/composables/useI18n`)

## Initialization Flow

```
1. App Bootstrap (main.ts)
   └── setupPlugins()             // Register i18n plugin
   └── getStoredLocale()          // Read from sessionStorage
   └── setI18nLanguage()          // Sync to vue-i18n

2. User Login (LoginView.vue)
   └── authStore.login()          // Authenticate
   └── mapBackendLanguageToLocale(userAuthInfo.language)
   └── persistLocale(locale)      // Update vue-i18n + sessionStorage
   └── Navigate to HOME

3. User Changes Language
   └── LanguageSwitcher or changeLocale()
   └── persistLocale()
   └── setI18nLanguage()
   └── HTML lang attribute updates
   └── All components re-render

4. App Reload
   └── getStoredLocale()          // Restore from sessionStorage
   └── setI18nLanguage()          // Continue with previous locale
```
